What working parallax actually uses

gentlerain.ai = GSAP + ScrollTrigger + Lenis. Here's what each piece does and why it matters.

Library 1

GSAP

Animation engine. Does the actual translateY transforms smoothly — frame-perfect, GPU-accelerated. Replaces our manual style assignment.

Library 2

ScrollTrigger

GSAP plugin that ties animation progress to scroll position. scrub: true means "at 50% scrolled = 50% animated." This is what makes layers move in sync with scroll — reliably, across all browsers.

Library 3

Lenis

Smooth scroll. Makes the native browser scroll feel buttery. Feeds its position to GSAP's ticker so ScrollTrigger reads smooth values — no jitter or lag.

The architecture that makes it work

  <div class="layer" style="position:absolute">trees</div>
  <div class="layer" style="position:absolute">fence</div>
</footer>

JS: fires only when footer enters viewport
Moves layers 20-80px inside a 420px box
→ Technically works but visually imperceptible
✅ What actually works
<!-- Layers fixed to VIEWPORT -->
<div class="sky"   style="position:fixed"></div>
<div class="trees" style="position:fixed"></div>
<div class="fence" style="position:fixed"></div>

<!-- Content scrolls OVER the layers -->
<main style="position:relative; z-index:10">...</main>

GSAP ScrollTrigger scrubs each layer's
translateY at a different rate as you scroll
→ sky barely moves, fence moves most
