Why our parallax isn't working — and how to fix it

Three techniques exist. We used the wrong one. Here's the difference.

The core problem

Our layers ARE stacked correctly (you confirmed position: absolute is applying). The issue is what moves them. Our JS only fires when the footer enters the viewport, moving things tiny distances inside a 420px box. That's not parallax — it's a local nudge. Real parallax requires layers fixed to the viewport, moving at different rates across the entire page scroll.

The three techniques — pick one

A

background-attachment: fixed

Pure CSS. One line. Background image stays fixed while content scrolls past it.

Pros

  • Dead simple
  • No JS

Cons

  • One layer only
  • No speed control
  • Broken on iOS Safari

❌ Can't tell a layered story

B

CSS perspective + translateZ

Set perspective: 1px on a scroll container. Elements at negative Z move slower (they're "further away").

Pros

  • Pure CSS
  • True physics model
  • Unlimited layers

Cons

  • Scroll must be on a div, not body
  • Scale() compensation required
  • Tricky z-index behaviour

⚠️ Works but complex to tune

C

position: fixed + JS depth multipliers ✦ recommended

Background layers are position: fixed (viewport-locked). JS reads scrollY and moves each layer by scrollY × depth — deeper layers move less.

Pros

  • Works on mobile
  • Full speed control per layer
  • Easy to tune
  • Reliable cross-browser

Cons

  • Requires JS
  • Content needs z-index management

✅ Right tool for a multi-layer story

How Technique C works — visually

Layer CSS position Movement at scrollY=500px 🌤 Sky position: fixed translateY(500 × 0.05) = 25px ← barely moves 🌲 Far trees position: fixed translateY(500 × 0.15) = 75px 🌲🌲 Near trees position: fixed translateY(500 × 0.35) = 175px 🪵 Fence position: fixed translateY(500 × 0.60) = 300px ← moves most 📄 Content position: relative (normal) translateY(500 × 1.0) = 500px ← scrolls normally

Result: as you scroll 500px down, the sky barely moves, trees drift up slowly, fence comes up faster. Your content (services, about, contact) floats OVER the forest at normal scroll speed. The depth gap between layers IS the parallax.
